---
title: Quickstart
description: "Install the SDK, create a client, and run your first memory roundtrip."
---

<Steps>
  <Step title="Install">
    <CodeGroup>
      ```bash npm
      npm install @deyta-ai/sdk
      ```

      ```bash bun
      bun add @deyta-ai/sdk
      ```

      ```bash pnpm
      pnpm add @deyta-ai/sdk
      ```

      ```bash yarn
      yarn add @deyta-ai/sdk
      ```
    </CodeGroup>
  </Step>

  <Step title="Create a client">
    ```ts
    import { Deyta } from "@deyta-ai/sdk";

    const deyta = new Deyta({ apiKey: process.env.DEYTA_API_KEY! });
    ```

    <Note>
      `baseUrl` defaults to `https://api.deyta.ai`. Override it for staging or
      self-hosted gateways.
    </Note>
  </Step>

  <Step title="Store a memory">
    ```ts
    const stored = await deyta.memory.remember({
      namespace_id: "ns_123",
      content: "The team standup is every Tuesday at 10am UTC.",
      title: "Standup time",
    });

    // stored: { document_id, chunks_created, entities_extracted, relationships_created }
    ```
  </Step>

  <Step title="Recall it">
    ```ts
    const result = await deyta.memory.recall({
      namespace_id: "ns_123",
      query: "when is the standup?",
    });

    // result.results[]: RecallMatch[]
    ```
  </Step>
</Steps>

## Next: stop repeating `namespace_id`

If you'll issue several operations against the same namespace, scope the client
once and drop the per-call `namespace_id`:

```ts
const ns = deyta.namespaces.scope("ns_123");

await ns.remember({ content: "..." });
await ns.recall({ query: "..." });
```

See [Scoping](/sdk/typescript/scoping) for the full sub-client surface.
