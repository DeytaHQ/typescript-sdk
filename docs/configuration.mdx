The `Deyta` constructor accepts a single `DeytaConfig` object.

```ts
import { Deyta } from "@deyta-ai/sdk";

const deyta = new Deyta({
  apiKey: process.env.DEYTA_API_KEY!,
  baseUrl: "https://api.deyta.ai",
  timeout: 30_000,
  retries: {
    maxRetries: 2,
    initialBackoffMs: 500,
    maxBackoffMs: 8_000,
    retryOn: [408, 429, 500, 502, 503, 504],
  },
  fetch: customFetch,
  logger: (event) => console.debug("[deyta]", event),
});
```

## Options

<ParamField path="apiKey" type="string" required>
  Your Deyta API key. Sent as `Authorization: Bearer <apiKey>` on every request.
</ParamField>

<ParamField path="baseUrl" type="string" default="https://api.deyta.ai">
  Override for staging, regional endpoints, or self-hosted gateways.
</ParamField>

<ParamField path="timeout" type="number" default="30000">
  Per-request timeout in milliseconds. Combined with caller-supplied
  `AbortSignal`s — either can abort the request.
</ParamField>

<ParamField path="retries" type="RetryConfig">
  Retry policy for idempotent methods. See [Retries](/sdk/typescript/retries).

  <Expandable title="RetryConfig">
    <ParamField path="maxRetries" type="number" default="2">
      Maximum retry attempts after the initial request.
    </ParamField>
    <ParamField path="initialBackoffMs" type="number" default="500">
      First backoff delay. Doubles on each subsequent retry, with up to 250ms
      of jitter, capped at `maxBackoffMs`.
    </ParamField>
    <ParamField path="maxBackoffMs" type="number" default="8000">
      Upper bound on backoff between retries.
    </ParamField>
    <ParamField path="retryOn" type="number[]" default="[408, 429, 500, 502, 503, 504]">
      HTTP status codes that trigger a retry. Network errors always retry.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField path="fetch" type="typeof fetch">
  Inject a `fetch` implementation. Useful for tests, custom runtimes, or
  polyfills. Defaults to `globalThis.fetch`.
</ParamField>

<ParamField path="logger" type="(event: SdkLogEvent) => void">
  Hook that receives `request`, `response`, `retry`, and `error` events. No-op
  by default. Useful for wiring Pino, Winston, or `console`.
</ParamField>

## Logger events

```ts
type SdkLogEvent =
  | { type: "request"; method: string; url: string; attempt: number }
  | { type: "response"; status: number; durationMs: number }
  | { type: "retry"; attempt: number; backoffMs: number; reason: string }
  | { type: "error"; error: Error };
```

```ts
const deyta = new Deyta({
  apiKey,
  logger: (event) => {
    if (event.type === "retry") {
      console.warn(
        `[deyta] retry attempt=${event.attempt} backoff=${event.backoffMs}ms reason=${event.reason}`,
      );
    }
  },
});
```

## User-Agent

Every request carries a `User-Agent` header of the form:

```
deyta-sdk/0.2.0 (bun/1.1.0; darwin)
deyta-sdk/0.2.0 (node/20.10.0; linux)
deyta-sdk/0.2.0 (browser)
```

It cannot be overridden — but per-call `headers` can add additional headers.
See [Cancellation and per-call options](/sdk/typescript/cancellation).
