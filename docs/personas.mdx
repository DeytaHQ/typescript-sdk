---
title: Personas
description: "Top-level personas that own a backing namespace and a Digor-built profile."
---

A persona is a top-level resource. Creating one auto-provisions a backing
namespace where memory and integrations land; the persona's `id` is the
underlying Digor `agent_id` and is the handle every other persona method uses.

## Create

```ts
const persona = await deyta.personas.create({
  subject: "Alice",
  external_reference_id: "user-abc",   // optional
  description: "demo persona",          // optional
});
```

The calling API key is auto-granted access to the new backing namespace, so
subsequent persona operations work without extra permission steps.

<Note>
  The returned record includes `namespaceId` — the backing namespace where you
  can `remember` / `recall` / `ask` directly using the standard memory
  endpoints.
</Note>

## List

```ts
// One page at a time
const { data, pagination } = await deyta.personas.list({ page: 1, page_size: 20 });

// Or walk every page automatically
for await (const p of deyta.personas.iterate({ page_size: 50 })) {
  console.log(p.id, p.subject);
}
```

The list is scoped to personas the calling API key has access to.

## Read (with Digor composite)

`get` returns the local record merged with Digor's composite document under a
`digor` envelope:

```ts
const result = await deyta.personas.get(persona.id);
// or by external reference:
const result = await deyta.personas.getByExternalRef("user-abc");

if (result.digor.available) {
  result.digor.data;   // identity, traits, episodes, peers, facets, providers, ...
} else {
  // Local record exists but the persona hasn't been built yet — call build().
}
```

| Outcome | Response |
|---|---|
| Persona exists, build complete | `{ ...persona, digor: { available: true, data: ComposedPersona } }` |
| Persona exists, no build yet | `{ ...persona, digor: { available: false } }` |
| Persona not found | `404 NOT_FOUND` (thrown as `DeytaError`) |
| Digor unreachable | `503` (thrown as `DeytaError`) |

## Update

```ts
await deyta.personas.update(persona.id, {
  description: "new copy",
  external_reference_id: null,    // null clears, omit leaves unchanged
});
```

`subject` is upstream-owned and cannot be changed via the gateway.

## Delete

```ts
await deyta.personas.delete(persona.id);
```

Cascades to the backing namespace and all of its connections, labels, and
grants.

## Build / status

```ts
const { build_id } = await deyta.personas.build(persona.id);  // HTTP 202

const { status, last_built_at } = await deyta.personas.status(persona.id);
// status: "building" | "ready" | "not_built"
```

`build()` is fire-and-forget; not idempotent. The gateway returns
`409 CONFLICT` if a build is already in flight — poll `status()` to follow
progress.
